cs registerTemplate

Registers an existing template into the CloudStack cloud.

Flags

– --account value - an optional accountName. Must be used with domainId.
– --bits value - 32 or 64 bits support. 64 by default (default: 0)
– --checksum value - the MD5 checksum value of this template
– --details value - Template details in key/value pairs.
– --displaytext value - [required] the display text of the template. This is usually used for display purposes.
– --domainid value - an optional domainId. If the account parameter is used, domainId must also be used.
– --format value - [required] the format for the template. Possible values include QCOW2, RAW, and VHD.
– --hypervisor value - [required] the target hypervisor for the template
– --isdynamicallyscalable value - true if template contains XS/VMWare tools inorder to support dynamic scaling of VM cpu/memory (default: unset)
– --isextractable value - true if the template or its derivatives are extractable; default is false (default: unset)
– --isfeatured value - true if this template is a featured template, false otherwise (default: unset)
– --ispublic value - true if the template is available to all accounts; default is true (default: unset)
– --isrouting value - true if the template type is routing i.e., if template is used to deploy router (default: unset)
– --issystem value - true if the template type is system i.e., if template is used to deploy system VM (default: unset)
– --name value - [required] the name of the template
– --ostypeid value - [required] the ID of the OS Type that best represents the OS of this template.
– --passwordenabled value - true if the template supports the password reset feature; default is false (default: unset)
– --requireshvm value - true if this template requires HVM (default: unset)
– --sshkeyenabled value - true if the template supports the sshkey upload feature; default is false (default: unset)
– --templatetag value - the tag for this template.
– --url value - [required] the URL of where the template is hosted. Possible URL include http:// and https://
– --zoneid value - [required] the ID of the zone the template is to be hosted on
– --debug, -d -
– --dry-run, -D -
– --region value, -r value -
